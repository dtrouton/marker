# md-mgr — Lightweight macOS Markdown Reader/Editor

## Overview

A native macOS desktop app for reading and editing markdown files. Optimized for quickly viewing rendered markdown (especially files generated by Claude Code and similar tools) and making small edits via a rich text WYSIWYG editor. Lightweight, fast, no bloat.

## Tech Stack

**SwiftUI Shell + WKWebView Editor (Hybrid approach)**

- **SwiftUI** for all native chrome: window management, sidebar file tree, tab bar, toolbar, dialogs, menu bar
- **WKWebView** hosts an embedded web-based markdown editor (Milkdown, built on ProseMirror)
- **Swift ↔ JS bridge** via WKWebView message handlers for communication between layers
- Uses system WebKit — no bundled browser engine, small binary size
- **Deployment target:** macOS 14 (Sonoma) or later — enables modern SwiftUI features (Observable macro, improved NavigationSplitView)
- **App display name:** "MD Mgr"

### Why this approach

Building a rich text markdown editor from scratch in AppKit is a massive undertaking. Web-based WYSIWYG markdown editors (Milkdown, Tiptap) are mature and battle-tested. By using SwiftUI for everything the user sees as "the app" (sidebar, tabs, toolbar, dialogs) and embedding the web editor only for the document area, the app feels fully native while leveraging the best editing libraries available.

## Architecture

```
┌─────────────────────────────────────────────┐
│  SwiftUI App Shell (macOS native)           │
│  ┌──────────┬──────────────────────────┐    │
│  │ Sidebar  │  Tab Bar                 │    │
│  │ (file    │  ┌────────────────────┐  │    │
│  │  tree)   │  │  WKWebView         │  │    │
│  │          │  │  (Milkdown editor  │  │    │
│  │          │  │   + rendered view) │  │    │
│  │          │  └────────────────────┘  │    │
│  └──────────┴──────────────────────────┘    │
│  Toolbar: search, export, view toggle       │
└─────────────────────────────────────────────┘
```

### Layer 1: SwiftUI Shell

Responsible for:
- Main window and window management
- Sidebar file tree with recursive directory display
- Tab bar with unsaved-changes indicators
- Toolbar (search, export, view controls)
- Native dialogs (Open Folder, Save, Export)
- macOS integration (Finder "Open With", drag-and-drop, dark/light mode)
- Menu bar with standard macOS keyboard shortcuts

### Layer 2: Swift ↔ JS Bridge

Communication via `WKScriptMessageHandler`:
- **Swift → JS:** Load file content, switch read/edit mode, trigger save, apply theme
- **JS → Swift:** Content changed, save requested, image pasted, link clicked

Messages are JSON-encoded. The bridge is thin — it passes content and commands, no business logic.

### Layer 3: Web Editor (embedded HTML/CSS/JS bundle)

Bundled in the app's resources. Two modes sharing one WKWebView:

- **Read mode (default):** Markdown rendered to styled HTML with proper typography, syntax-highlighted code blocks, table rendering
- **Edit mode:** Milkdown WYSIWYG editor with formatting toolbar and keyboard shortcuts

## Features

### File Opening

- **From Finder:** App registers as handler for `.md` and `.markdown` files. Double-click or "Open With" opens the file directly in a new tab. No folder/sidebar required.
- **From sidebar:** Open a folder via `NSOpenPanel` or drag-and-drop to browse a file tree. Click a file to open in a new tab.
- **Drag-and-drop:** Drop `.md` files onto the app icon or window to open them.
- If a file opened from Finder is inside an already-opened folder, the sidebar context is preserved.

### Sidebar File Tree

- SwiftUI `List` with `OutlineGroup` for recursive directory display
- Live-updating via `DispatchSource` file system watcher (reflects external changes)
- Filters to `.md` and `.markdown` files by default
- Remembers last-opened folder between launches (via `UserDefaults` or `@AppStorage`)

### Tab Bar

- Each tab tracks: file path, dirty state (unsaved changes dot), scroll position
- Keyboard shortcuts: Cmd+W close, Cmd+Shift+] / Cmd+Shift+[ to switch tabs
- Middle-click or X button to close
- Closing a tab with unsaved changes prompts to save
- New tabs open to the right of the current tab

### Read Mode

- Default mode when opening any file
- Markdown rendered to styled HTML
- Syntax highlighting for code blocks (via highlight.js or Prism embedded in the web bundle)
- Proper rendering of tables, block quotes, task lists, front matter
- Images displayed (relative paths resolved to the file's directory)
- Scroll position preserved per tab

### Edit Mode

- Activated by: clicking "Edit" button in toolbar, double-clicking the rendered content, or pressing Enter/Return
- Milkdown WYSIWYG editor (ProseMirror-based, markdown-native)
- Formatting toolbar: bold, italic, headings (H1-H6), bullet/numbered lists, links, code (inline + blocks), block quotes, horizontal rules
- Keyboard shortcuts: Cmd+B bold, Cmd+I italic, Cmd+K insert link, Cmd+Shift+C code block
- Paste image support: saves image to a relative path alongside the file, inserts markdown image reference
- Escape or "Done" button returns to read mode
- Content round-trips cleanly between markdown source and WYSIWYG (Milkdown's core design)

### Search

- **In-file (Cmd+F):** Find and replace within the current document. Handled in the WKWebView layer.
- **Folder-wide (Cmd+Shift+F):** Full-text search across all `.md` files in the opened folder. Uses `NSMetadataQuery` (Spotlight index) for performance. Results displayed in a panel that replaces the sidebar. Clicking a result opens the file and scrolls to the match.

### Export

- **Cmd+E** opens an Export dialog with format options (PDF or HTML)
- **PDF:** Uses WebView's print-to-PDF with the rendered HTML. Styled output with no extra work.
- **HTML:** Standalone `.html` file with embedded styles.
- Accessible via File > Export menu.

### System Integration

- **Dark/light mode:** SwiftUI follows system appearance. Web editor CSS uses `prefers-color-scheme` media query.
- **Spotlight indexing:** File type registration enables Spotlight to index `.md` files opened with the app.
- **Quick Look extension:** Included in the app bundle so `.md` files render as formatted markdown in Finder's preview pane and Quick Look (spacebar).

## Data Model

No database. The app operates directly on the file system.

- **AppState:** Currently opened folder path, list of open tabs, active tab index, search state
- **TabState:** File URL, markdown content (String), dirty flag, scroll position, current mode (read/edit)
- **Persistence:** Last-opened folder and window frame stored in `UserDefaults`. No other persistent state.

## Error Handling

- File deleted while open: Show inline banner "File was deleted or moved", allow saving to new location
- Permission denied: Show native alert, suggest re-opening with permissions
- Large files (>1MB): Load and render normally, but warn before entering edit mode if >5MB
- Malformed markdown: Render best-effort (markdown renderers are forgiving by design)

## Out of Scope

- Version control / git integration
- Markdown templates or snippet libraries
- Cloud sync
- Plugin system
- Multiple windows (single window, multiple tabs)
- Vim/Emacs keybindings
- Custom themes (follows system appearance only)
